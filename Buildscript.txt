Now, there are gonna be the prebuilt binaries in the release section of the repo. (usually)
So id advise for you people to compile the code yourself.
But, if one wants to modify the codebase, and run out their own version.
I strongly, and i say, strongly advise to use the zig build system. (can be done by just installing Zig system wide).

Here are the following versions, otherwise sound and other things wont play.

*FOR LINUX DISTROS:*
zig build-exe FileName.zig -lc -target x86_64-linux-musl -O ReleaseFast

*FOR freeBSD*
zig build-exe FileName.zig -target x86_64-freebsd -O ReleaseFast


*FOR macOs* (UNTESTED)

zig build-exe FileName.zig -target x86_64-macos -O ReleaseFast
zig build-exe FileName.zig -target aarch64-macos -O ReleaseFast
